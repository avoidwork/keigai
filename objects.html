Core Objects

keigai has a minimal interface, exposing four types of Objects for use:

DataStore

DataStores are in ram NoSQL databases, with many features found in SQL databases, such as JOINs, ORDER BYs, & WHERE clauses. DataStores use Web Workers for "sorting" & "selecting" records, when possible, to avoid blocking the UI event loop.

DataStores can be wired to an API through the setUri() method, which hooks them into the feedback loop. If you "set" or "delete" a record, the operation doesn't complete until the API provides a "success" or "failure" response.

Deferreds (Promises) are returned from methods that can "go over the wire", for easy handling of the asynchronous communication.

Options

autosave (Boolean)    Enables persistent storage to localStorage, sessionStorage or MongoDB (on the server)
callback (String)     JSONP facing variable to denote the callback variable in the request URL
credentials (Boolean) Sets withCredentials on XHR for API interaction (default is null)
depth (Number)        Current API crawling depth relative to 'origin' DataStore
events (Boolean)      Enable/disable Observer based pub/sub behaviour (default is true)
expires (Number)      Milliseconds, (default is null)
headers (Object)      Outbound headers to decorate on the XHR
ignore (Array)        Custom properties to not send back an API, e.g. UI facing settings on records
key (String)          Primary key for records (default is null, generates a UUID v4)
leafs (Array)         Keys to treat as leafs, e.g. do not crawl them
maxDepth (Number)     Maximum depth to travese during API crawling
mongodb (String)      MongoDB connection string for persistent storage when autosave is true
retrieve (Boolean)    Triggers automatic crawling of API responses
source (String)       API response property that contains records, e.g. "data" in {"data": [...]}
versioning (Boolean)  Enable/disable record versioning (default is true)
uri (String)          API end point (default is null)

DataGrid

DataGrids combine DataStores, DataLists, & DataListFilters into fast & responsive tabular representations. DataGrids can be sorted by many "fields" in different directions, similar to an SQL ORDER BY clause.

DataGrids differ from many table based solutions, by only rendering what can be seen. This minimizes DOM repaints/reflows, and keeps the UI fast.

API documentation

DataList

DataLists are the visual representation of a DataStore, using "handlebar" style templates. They can be sorted, paginated, & styled with CSS. DataLists can be identified with the "list" CSS class on their UL Element.

DataLists bind to a DataStore, and become a reactive view, refreshing automatically when the DataStore records change.

API documentation

DataListFilter

DataListFilters provide a fast & easy way for the end user to refine the contents of a DataList or DataGrid, by binding to an existing DOM input Element. Wildcards & regular expressions are supported. Keyboard input is debounced to avoid excessive filtering.
