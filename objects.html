DataStore

keigai.store([data, options])

DataStores are in ram NoSQL databases, with many features found in SQL databases, such as JOINs, ORDER BYs, & WHERE clauses. DataStores use Web Workers for "sorting" & "selecting" records, when possible, to avoid blocking the UI event loop.

DataStores can be wired to an API through the setUri() method, which hooks them into the feedback loop. If you "set" or "delete" a record, the operation doesn't complete until the API provides a "success" or "failure" response.

Deferreds (Promises) are returned from methods that can "go over the wire", for easy handling of the asynchronous communication.

API documentation

Options

  • autosave Boolean
    Enables persistent storage to localStorage, sessionStorage or MongoDB (node.js)
  • callback String
    JSONP callback variable, will be decorated in request URL
  • credentials Boolean
    Sets withCredentials on XHR for API interaction (default is null)
  • depth Number
    Current API crawling depth relative to 'origin' DataStore
  • events Boolean
    Enable/disable Observer based pub/sub behaviour (default is true)
  • expires Number
    Explicit expiration in milliseconds, triggers a re-sync with the API (default is null)
  • headers Object
    HTTP request headers
  • ignore Array
    Custom keys not to be sent back an API, e.g. UI facing settings on records
  • key String
    Primary key for records (default is null, generates a UUID v4)
  • leafs Array
    Keys to treat as leafs; they will not be crawled
  • maxDepth Number
    Maximum depth to travese during API crawling
  • mongodb String
    MongoDB connection string for persistent storage when autosave is `true`, and running in node.js
  • retrieve Boolean
    Triggers automatic crawling of API responses
  • source String
    API response property that contains records, e.g. "data" in `{"data": [...]}`
  • versioning Boolean
    Enable or disable record versioning (default is true)
  • uri String
    API end point (default is null)

DataGrid

keigai.grid(target, store, fields[, sortable, options, filtered, debounce])

DataGrids combine DataStores, DataLists, & DataListFilters into fast & responsive tabular representations. DataGrids can be sorted by many "fields" in different directions, similar to an SQL ORDER BY clause.

DataGrids differ from many table based solutions, by only rendering what can be seen. This minimizes DOM repaints/reflows, and keeps the UI fast.

API documentation

Options

  • callback Function
    Callback function to execute on DataGrid rows, argument is the new Element
  • emptyMsg String
    Message to display when there is no records to display
  • pageIndex Number
    Page displayed
  • pageRange Number
    Range of pages to display (default is 5)
  • pageSize Number
    Page size (default is 5)
  • pagination String
    Pagination position: "top", "bottom", or "top|bottom" (default is "bottom")
  • placeholder String
    String to use in lieu of an undefined record (data) property
  • order String
    SQL `ORDER BY` clause for grid columns (default is null)
  • where Number
    Maximum depth to travese during API crawling

DataList

keigai.list(target, store, template[, options])

DataLists are the visual representation of a DataStore, using "handlebar" style templates. They can be sorted, paginated, & styled with CSS. DataLists can be identified with the "list" CSS class on their UL Element.

DataLists bind to a DataStore, and become a reactive view, refreshing automatically when the DataStore records change.

API documentation

Options

  • callback Function
    Callback function to execute on DataList rows, argument is the new Element
  • emptyMsg String
    Message to display when there is no records to display
  • filter String
    Comma delimited string of keys to filter on
  • pageIndex Number
    Page displayed
  • pageRange Number
    Range of pages to display (default is 5)
  • pageSize Number
    Page size (default is 5)
  • pagination String
    Pagination position: "top", "bottom", or "top|bottom" (default is "bottom")
  • placeholder String
    String to use in lieu of an undefined record (data) property
  • order String
    SQL `ORDER BY` clause for grid columns (default is null)
  • where Number
    Maximum depth to travese during API crawling

DataListFilter

keigai.filter(target, list, filters[, debounce])

DataListFilters provide a fast & easy way for the end user to refine the contents of a DataList or DataGrid, by binding to an existing DOM input Element. Wildcards & regular expressions are supported. Keyboard input is debounced to avoid excessive filtering.
