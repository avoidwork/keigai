DataStore

keigai.store([data, options])

DataStores are in ram NoSQL databases, with many features found in SQL databases, such as JOINs, ORDER BYs, & WHERE clauses. DataStores use Web Workers for "sorting" & "selecting" records, when possible, to avoid blocking the UI event loop.

DataStores can be wired to an API through the setUri() method, which hooks them into the feedback loop. If you "set" or "delete" a record, the operation doesn't complete until the API provides a "success" or "failure" response.

Deferreds (Promises) are returned from methods that can "go over the wire", for easy handling of the asynchronous communication.

API documentation

Options

  • autosave Boolean
    Enables persistent storage to localStorage, sessionStorage or MongoDB (node.js)
  • callback String
    JSONP callback variable, will be decorated in request URL
  • credentials Boolean
    Sets withCredentials on XHR for API interaction (default is null)
  • depth Number
    Current API crawling depth relative to 'origin' DataStore
  • events Boolean
    Enable/disable Observer based pub/sub behaviour (default is true)
  • expires Number
    Explicit expiration in milliseconds, triggers a re-sync with the API (default is null)
  • headers Object
    HTTP request headers
  • ignore Array
    Custom keys not to be sent back an API, e.g. UI facing settings on records
  • key String
    Primary key for records (default is null, generates a UUID v4)
  • leafs Array
    Keys to treat as leafs; they will not be crawled
  • maxDepth Number
    Maximum depth to travese during API crawling
  • mongodb String
    MongoDB connection string for persistent storage when autosave is `true`, and running in node.js
  • retrieve Boolean
    Triggers automatic crawling of API responses
  • source String
    API response property that contains records, e.g. "data" in `{"data": [...]}`
  • versioning Boolean
    Enable or disable record versioning (default is true)
  • uri String
    API end point (default is null)
